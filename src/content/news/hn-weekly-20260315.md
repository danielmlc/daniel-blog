## Baochip-1x：bunnie Huang 的开源 RISC-V 芯片，带 MMU

**评分：262 · 49 条讨论**

硬件黑客先驱 Andrew "bunnie" Huang 在 Crowd Supply 上宣布 Baochip-1x，一款采用 22nm 工艺、搭载内存管理单元（MMU）的 RISC-V 微控制器。MMU 在同级嵌入式系统中极为罕见，它意味着可以在裸机上运行 Linux 完整内核，而非仅限于 RTOS。更值得关注的是：该芯片通过与 Crossbar 公司合作，利用其现有硅片上的闲置面积完成，大幅压低了定制芯片的成本门槛。bunnie 将其描述为"向开源计算未来迈进的务实一步"，RTL 代码将以"大部分开放"的形式发布。

→ [Crowd Supply 项目页](https://www.crowdsupply.com/baochip/dabao/updates/what-it-is-why-im-doing-it-now-and-how-it-came-about)

## Python 性能优化阶梯：从 1.4× 到 520× 的全景基准测试

**评分：258 · 93 条讨论**

一篇详尽的基准评测文章，将 Python 性能优化路径按递进层次排列：升级 CPython 版本（约 1.4×）→ 使用 NumPy 向量化（约 520×）→ Cython 或 Rust 扩展（99–154×）。作者揭示了一个核心结论：Python 的慢根源于"极度动态"的设计，每次操作都需要运行时分发，而非仅仅是 GIL 或解释器开销。"优化的努力曲线是指数级的"——这意味着大多数项目在 NumPy 这一层就已足够，只有真正的性能瓶颈才值得跨越到编译方案。

→ [原文](https://cemrehancavdar.com/2026/03/10/optimization-ladder/)

## 蒙大拿州通过"计算权"法案，成全美首例

**评分：234 · 202 条讨论**

蒙大拿州（Montana）立法机构通过《计算权利法案》（Right to Compute Act），正式将个人在家中运行自有硬件和软件的权利写入州法律，禁止政府和运营商以"合规"或"许可"为由阻止公民在自己的设备上运行任意代码。该法案诞生于对云服务垄断和运营商锁定趋势的强烈反弹，也被视为对近年来各地推进"受信任计算"监管潮流的直接回应。HN 讨论中，支持者将其视为数字权利的里程碑，反对者则担忧会为恶意软件部署提供法律庇护。

→ [西部蒙大拿新闻报道](https://www.westernmt.news/2025/04/21/montana-leads-the-nation-with-groundbreaking-right-to-compute-act/)

## XML 是一种廉价 DSL——被低估的声明式语言载体

**评分：231 · 239 条讨论（本周最热讨论）**

本周讨论量最大的文章，作者为 XML 正名：在构建领域专用语言（DSL）时，XML 相比 JSON 或自制语法往往是更廉价、更务实的选择。以税务计算引擎为例，XML 格式的规则声明可以直接被 XPath/XSLT 工具链审计和查询，而无需额外开发调试基础设施。核心论点是：XML 的"冗长"换来了通用生态系统的全套免费工具，这笔账在企业级系统中往往是合算的。评论区对此分歧明显，但整体质量很高。

→ [原文](https://unplannedobsolescence.com/blog/xml-cheap-dsl/)

## Ageless Linux：一个拒绝遵守年龄验证法的 Linux 发行版

**评分：144 · 99 条讨论**

Ageless Linux 是一个基于 Debian 的 Linux 发行版，其创作动机是对抗加利福尼亚州 AB 1043 年龄验证法。项目通过修改 `/etc/os-release` 将自身注册为"受监管的操作系统提供商"，随后故意拒绝收集用户年龄数据——以一种荒诞而精准的方式实施公民不服从。作者认为，此类立法以"儿童安全"之名强制推行监控基础设施，本质上是对所有用户隐私的侵害。这个项目本身更像一件政治艺术品，但其引发的法律和政策讨论相当严肃。

→ [Ageless Linux 官网](https://agelesslinux.org/)

## Jazzband 宣布关闭：AI 垃圾代码攻破了开源协作模型

**评分：132 · 44 条讨论**

Jazzband 是一个运行超过十年的开源协作组织，其核心理念是给所有成员共享推送权限，以降低维护 71 个 Python 项目的负担。然而创始人宣布，AI 生成的垃圾 PR 和贡献洪流已经使这种开放信任模型彻底不可持续——审核成本远超协作收益。同时，项目也暴露出治理结构的单点故障问题：创始人长期是唯一的"关键人"，没有建立起可持续的接班机制。这是 AI 规模化生成代码对开源社区造成具体伤害的又一典型案例。

→ [Jazzband 关闭公告](https://jazzband.co/news/2026/03/14/sunsetting-jazzband)

## OpenTTD 在 Steam 上停止单独销售，捆绑进复刻版合集

## Megadev：Sega Mega Drive 和 Mega CD 的现代开发工具包

**评分：116 · 7 条讨论**

一个为世嘉 Mega Drive（即 Genesis）和 Mega CD 硬件打造的完整现代开发套件，提供 C 语言 API、调试工具链和文档体系，让开发者无需深入汇编即可开始 MD 平台开发。在复古游戏开发（Homebrew）社区持续升温的背景下，这类高质量工具的出现进一步降低了经典平台创作的门槛。项目托管于 GitHub，完全开源。

→ [GitHub 仓库](https://github.com/drojaazu/megadev)
