**项目概述：**

`typescript-design` 是开源在 GitHub 上的一个通过 TypeScript 实现经典设计模式（Design Patterns）的工程化代码库。本项目的初衷是为了将抽象的软件工程架构思想，利用具备强类型特性的 TypeScript 语言，通过极其生动和详细的核心示例进行展示与解释，使得开发者能够更快速地掌握面向对象架构的精髓。

如果你正在学习或重构软件代码的模块化设计，这将是一份不错的代码实践指南！

## 设计模式目录

项目中系统化地收录并用代码实现了以下四大类设计模式：

### 1. 创建型模式 (Creational Patterns)
这部分模式提供单例、实例构建和解耦实例化的底层实现机制：
- **工厂方法模式 (Factory Method)**
- **单例模式 (Singleton)**

### 2. 结构型模式 (Structural Patterns)
这类模式处理类或对象如何组合成更大的结构，同时保持结构的灵活性和高效性：
- **适配器模式 (Adapter)**
- **组合模式 (Composite)**
- **装饰器模式 (Decorator)**
- **外观模式 (Facade)**
- **代理模式 (Proxy)**

### 3. 行为型模式 (Behavioral Patterns)
主要处理对象之间的通信、状态变化机制等高级交互：
- **命令模式 (Command)**
- **迭代器模式 (Iterator)**
- **观察者模式 (Observer)**
- **状态模式 (State)**
- **策略模式 (Strategy)**
- **模板方法模式 (Template Method)**

### 4. 复合模式 (Compound Patterns)
- **复合模式 (鸭子模拟器)** —— 结合了多种底层模式处理复杂场景。
